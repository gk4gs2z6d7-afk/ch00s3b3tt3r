Structural Humanism nav fixes

This file gives you copy-paste nav examples to fix broken links for /case-files and /signal-news.

What is likely wrong

  • If your file is named case-files.html but the nav links to /case-files, the link can 404 on a static site.
  • If your file is named signal-news.html but the nav links to /signal-news, same problem.
  • Static HTML sites should usually link directly to the actual filename unless you created folder routes like case-files/index.html.

Use this if your files are named .html

<nav class="site-nav" aria-label="Primary">
  <a href="/">Home</a>
  <a href="/case-files.html">Case Files</a>
  <a href="/signal-news.html">Signal News</a>
  <a href="/works/">Works</a>
  <a href="/#contact">Contact</a>
</nav>

Use this if you made folder routes

case-files/index.html
signal-news/index.html

Then your nav can be:

<nav class="site-nav" aria-label="Primary">
  <a href="/">Home</a>
  <a href="/case-files">Case Files</a>
  <a href="/signal-news">Signal News</a>
  <a href="/works/">Works</a>
  <a href="/#contact">Contact</a>
</nav>

Recommendation

Fastest fix: if you already have case-files.html and signal-news.html, change the nav to point to those exact filenames.

Checklist

  • Confirm the repo contains case-files.html and/or signal-news.html.
  • Update every nav link, button, and footer link to the same exact path.
  • Push to GitHub and wait for Vercel to redeploy.
  • Test both /case-files.html and /signal-news.html directly.

Also likely needed for Signal News

If the page loads but shows no news, fixing the nav will only get you to the page. You may still need to add actual news items to the page content.
